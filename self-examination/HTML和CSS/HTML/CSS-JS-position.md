## CSS 和 Javascript 引入放在 HTML 的位置及原因

在 HTML 文档中，CSS 和 Javascript 文件的防治位置会影响到页面加载和渲染的性能

### CSS 的放置位置

推荐放置位置：在 `<head>` 标签内

原因：

- 渲染阻塞：浏览器在渲染页面时，会阻塞渲染直到所有 CSS 被解析和应用，以避免渲染闪烁或不正确的布局。将 CSS 放在 `<head>` 中可以确保在页面内容渲染之前，CSS 已经被加载和解析，从而提高用户体验

- 提前加载：将 CSS 放在 `<head>` 中，可以让浏览器尽早地发现和加载 CSS 资源，减少页面渲染的等待时间

### Javascript 的放置位置

推荐放置位置：在 `<body>` 标签的底部，即在关闭的 `</body>` 标签之前

原因：

- 避免渲染阻塞：Javascript 文件的加载和执行可能会阻塞 HTML 的渲染和解析。将 Javascript 放在底部可以确保在所有 HTML 内容被解析和显示之后再加载执行脚本，减少对渲染过程的干扰

- 提高性能：将脚本放在底部可以提高页面的加载速度，因为这样做允许浏览器先加载页面的可视内容，从而使用户尽快看到页面内容

### 使用异步或延迟加载的脚本

对于那些不需要在页面加载时立即执行的 Javascript，可以通过 `async` 或 `defer` 属性来优化其加载方式

- `async` 属性：适用于那些不依赖于其他脚本且其他脚本也不依赖于它的脚本。它允许脚本异步加载，一旦下载完成立即执行，但不保证脚本执行的顺序

```html
<script async src="script.js"></script>
```

- `defer` 属性：适用于那些需要在文档完全解析后才能运行的脚本。带有 `defer` 的脚本会按照它们在文档中出现的顺序执行，且是在 DOM 解析完成后、`DOMContentLoaded` 事件触发之前执行

```html
<script defer src="script.js"></script>
```
